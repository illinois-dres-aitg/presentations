Color Contrast: SVG and High Contrast Mode

When an SVG image is used as a user interface component, if you specify the foreground color (usually 'stroke' or 'fill') as 'currentColor', it will inherit its color value from its nearest ancestor (in our case, the 'button' element).

The utility of doing this pays off in high contrast mode, which is controlled at the operating system level.

If HCM inverts the colors on a page, [for example from 'white foreground on dark background' to 'dark foreground on white background',] it will change the 'button' foreground and background colors, but it does not 'tamper' with the internal settings of SVG images.

Because 'currentColor' was used, the SVG image still renders correctly.
