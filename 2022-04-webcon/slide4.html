Keyboard Support: Examples

Test page for exploring the visual differences between passive and active focus styling.

Web Page Sequential Navigation and Activation Focus: Default Focus: Active Outline Notes
Fighting Illini Home Page no Yes no
  • Video controls are not keyboard accessible.
  • Poor focus styling, relies on default focus styling with mixed results.
Zoom Home Page no Yes Yes
  • Keyboard activation for carousels operates on the second carousel and there is not sequential access to the carousel controls on the second carousel .
  • Poor focus styling, inconsistent used of active and default focus styling creates missing or ambivelent focus indication.
DRES Accessible IT Group Yes no Yes
  • All links and controls can be accessed through the keyboard.
  • Keyboard activation and pointer "click" on items have same results.
  • Actively style keyboard focus to provide an outline around links and form controls.
Rutgers University home page Yes no Yes
  • Sub-menu open/close controls not easy to discerne.
  • If search opened, it is not automatically closed and obscures content.
  • Generally good keyboard focus styling with use of a outline.
Illinois Web Theme Yes no Yes
  • Some keyboard enhancements result in lost focus, using arrow keys when a menu is open.
  • Poor keyboard focus, relies mostly on color changes and no outlining.
