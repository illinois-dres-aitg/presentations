Test Page: role and aria-checked

Accessibility problems exist when return values are "undefined" or the return type is not the same on every browser. developers may make assumptions on the consistency of using the IDL to get ARIA roles, porperties and states.

Getting value with .role property
Setting Code Return value of .role
<input type="checkbox"/>
<div role="checkbox">
e.setAttribute('role', 'checkbox');
e.role = 'checkbox';
Getting value of .ariaChecked property
Setting Code Return value of .ariaChecked typeof.ariaChecked
<div role='checkbox' aria-checked='true'>
e.setAttribute('aria-checked', 'true');
e.ariaChecked = 'true'; (String)
e.ariaChecked = true; (Boolean)
